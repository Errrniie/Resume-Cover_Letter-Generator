# Job-Type Curation Guide

For each engineering discipline, this file specifies how to frame projects, reword work experience, order skills, and use discipline-specific terminology.

---

# Required Rule

ALL engineering projects and work experience sections must contain at least 2 bullet points.

NEVER omit a project or work experience section entirely.

If a project or work experience section is less relevant to the role, still tailor the language and keywords to align it as closely as possible with the job description.

 When tailoring bullets, prioritize relevant metrics from the Resume Reference Material when they strengthen the job alignment. Do not include metrics that distract from the primary focus of the role.

## Key Terminology by Discipline

| Discipline | Preferred Language | Avoid |
|---|---|---|
| **Manufacturing** | Machine parameters, process control, part processing, quality inspection, reliability | Real-time detection, vision algorithms |
| **Embedded Systems** | Microcontroller, firmware, real-time, hardware-firmware integration, sensor fusion | Aerodynamics, flow control |
| **Aerospace** | Aerodynamic analysis, ANSYS, FEA, boundary layer, flow control, wind tunnel | Motor control, real-time systems |
| **Systems Engineering** | Architecture, integration, cross-functional, end-to-end, subsystems, communication | Narrow technical focus |
| **Electrical** | Power systems, circuit design, inverter, power management, custom fabrication | High-level descriptions |
| **Leadership** | Leadership, operations, management, stakeholder, coordination, budget | Technical depth over vision |
| **PCB Design** | PCB design, circuit board, schematic, custom fabrication, board layout | Software-centric language |
